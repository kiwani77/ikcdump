🔥 CMDB Exam Final Check

1. 헷갈리는 대시보드 3대장

📍 3C (Correctness, Compliance, Completeness) → Health Dashboard
📍 Playbook, Best Practice, IRE 사용률 → Data Foundations Dashboard
📍 CSDM 도메인 간 연결 상태 → CSDM Data Foundations Dashboard

* "Actionable", "Playbook" 단어 보이면 무조건 Data Foundations!

2. CSDM 도메인 & 매칭 공식

도메인핵심 CI / 키워드
DesignBusiness App, Information Object (PII/PCI)
Manage TechnicalApp Service, Technical Service, Infra CI
Sell/ConsumeBusiness Service, Service Offering, Consumer
FoundationLocation, User, Group, Managed by group

* "EA(Architect)가 일하는 곳?" → Design

3. 묻지도 따지지도 않는 매칭 (1초 컷)

  • Attestation = 존재 여부 (Still exists?)
  • Data Certification = 필드 값 검증/수정 (Accuracy)
  • Information Object = 민감 데이터 (PII, PCI, Privacy)
  • CMDB Group = Dashboard Filter, 특정 Subset
  • Life cycle mappings = Legacy Status → CSDM 표준 변환

4. 데이터 관리 & Cleanup

📍 삭제/보관 정책 기반 자동화 → CMDB Data Manager
📍 중복 CI 해결 (Remediation) → Duplicate CI Remediator (in Workspace)
  • Archive: 보관 (복구 가능)
  • Delete: 영구 삭제 (공간 확보)
  • Target: Retired 또는 End of Life 상태의 CI

5. 신규 CI 클래스 생성 Rule

1️⃣ Store App (Class Models)에서 먼저 찾기 (중복 방지)
2️⃣ 없으면 기존 CI Class를 Extend(상속) 하기
⚠️ Asset 클래스를 확장한다는 보기는 오답!

6. IRE (식별 & 조정 엔진)

  • 어디서 설정? CI Class Manager
  • 우선순위(Reconciliation): 숫자가 작을수록 높은 우선순위!
  • 외부 데이터 Import 시: CMDBTransformUtil API 사용 (IRE 강제)

7. 관계(Relationship) 유형

  • Many-to-Many: Load Balancing, Redundancy (이중화)
  • One-to-Many: 하나의 서버에 여러 앱 (단순 호스팅)
  • 조회 도구: Query Builder (Non-CMDB 테이블인 Incident 등도 연결 가능)
